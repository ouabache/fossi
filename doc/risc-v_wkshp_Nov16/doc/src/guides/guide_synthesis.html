Synthesis Requirements



Use ONLY Synchronous design methodology

1)  No Latches.

2)  All flipflops must be rising edge triggered on the same clock.

3)  No Asynchronous resets on flipflops. You can have a asynchronous reset system BUT you must build it using synchronous resets.

4)  No # delays at all. Especially on the RHS of a non-blocking statement.  They really don't do what you think they are doing.

5)  All non-synthesisable code must be in a `ifndef SYNTHESIS block

6)  Do not `include any file that is not in the rtl/verilog directory

7)  multicycle paths must have an associated clock enable signal and only update on that signal

8) clock enable signals must be treated as a seperate clock domain and all input signals resynced to that domain.

    





Design for verification

1)  All flipflops must have a power on reset.

2)  Use multicycle resets.
