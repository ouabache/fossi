Software views editor

This is Kactus extension


SW views are usually associated with a CPU component. They define which application-independent SW, such as operating system or device drivers, are available. User selects which SW view is active, e.g. uCos, uCLinux or not OS at all, for each CPU In system design. The structure of a view is captured with SW design which also a Kactus-extension and saved in its own XML file.

Each SW view has mandatory name and hierarchy reference. Note that the latter cannot be edited here, but in view's own editor. Note also that a SW view is always hierarchical, unlike regualr view.Display name is optional and typically a few words providing a more detailed and/or user-friendly name for display. Optional description contains a textual description of the interface.

EXAMPLE. GizmoCPU has a software stack that has three components: DMA driver and two components for implementing MCAPI. The SW view points to s SW design which instantiates and connects these three together. Note that the same SW structure could be captured also in system design, but doing it like this simplifies reuse. SW views becomes especially handy when GizmoCPU supports multiple SW stacks.
