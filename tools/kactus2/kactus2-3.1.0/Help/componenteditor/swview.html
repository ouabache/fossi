Software view editor

This is Kactus extension


SW view is usually associated with CPU components and defines which application-independent SW, such as operating system or device drivers, are available. User selects which SW view is active, e.g. uCos, uCLinux or not OS at all, for each CPU In system design.

Each SW view has a mandatory name. Display name is optional and typically a few words providing a more detailed and/or user-friendly name for display. Optional description contains a textual description of the interface.

Hierarchy reference is the VLNV of SW design which also a Kactus-extension. It captures the structure of a SW stack and is saved in its own XML file. You can type the VLNV or drag-drop a SW design from the library.
