View editor

View specifies a representation level of a component, e.g. there might be different views for simulation and synthesis. This editor is used to edit the details of single view.

View has name and textual description. Display name can be more detailed or user-friendly than name, but usually only few words.

Environment identifiers in mandatory textual information about how this view is deployed in different tool environments.

  • Language indicates this view may be compatible with a particular tool, but only if that language is supported in that tool, e.g., different versions of some simulators may support two or more languages.
  • Tool indicates that this view contains tool-information, e.g., HDL models that use simulator-specific extensions. The tool can have a generic value such as "*Simulation" and "*Synthesis".
  • VendorSpecific can further qualify tool and language compatibility, e.g. if some appropriate simulator mode must be enabled.

Instantiations

Component instantiation specifies the referenced component instantiation. Data regarding the selected component instantiation is displayed in the view editor. For more information on the component instantiation, see the component instantiation editor.

Design configuration instantiation specifies the referenced design configuration instantiation. Data regarding the selected design configuration instantiation is displayed in the view editor. For more information on the design configuration instantiation, see the design configuration instantiation editor.

Design instantiation details the referenced design instantiation. Data regarding the selected design instantiation is displayed in the view editor. For more information on the design instantiation, see the design instantiation editor.

Module parameters

Module parameters are optional elements that configure the model for this view. In IP-XACT, module parameters are an element of the component instantiation. For convinience, they can be viewed in the view editor. The module parameters can be utilized as view-specific parameters. For further information on module parameters, see the component instantiation editor.
