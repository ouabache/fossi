File editor

File editor can be used to edit the details of a single file within a file set.

Name field contains the relative file path from the containing component's xml-file to this file. The name cannot be changed after creating a file item.
Description shows a used given description regarding the currently active file.
File types list specifies the type of the file. File type can be used i.e. when specifying build commands for a group of files. A file can have multiple file types.

General options

Logical name specifies e.g. a VHDL library where the VHDL-file will be compiled to.
If the checkbox generators can override logical name is ticked, the logical name can be overridden by another process. If the check box is unticked, then this logical name cannot be overrriden.
The check box File is structural RTL informs that the file is a structural register transfer level file.
The check box file is include file can be used to specify the file is an include file.
The check box file contains external declarations can be used to specify that the file contains external declarations and is needed by other files in this file set.

Build command

Target file specifies a path to the file that is derived from this file when build process is run.
Build command can be used to define a specific build command and flags for this file.
Flags can either replace default flags (e.g. those specified for this file type) in the build script or be appended to them. The value for replace default flags can be given as an expression, but must evaluate to 1 or 0.

External dependencies and defines

Dependencies contains a list of paths to directories containing (include) files on which the file depends.
Exported names contains a list defining any names that can be referenced externally.
Image types contains a list of the current files relations to executable image types in the design.
