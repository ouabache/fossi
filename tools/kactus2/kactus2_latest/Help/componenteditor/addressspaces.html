Address spaces editor

Address spaces editor contains the summary of the address spaces of the component. This editor can be used to add and remove address spaces to the containing component. Double-click adds a new address space.
In the elements, f(x) means that the value can be given as an expression.

Address space defines the addressable memory space that can be accessed via a master interface. It is the programmer's view to the rest of the system.
Note that address spaces are actually instance-specific concepts (what other units are there and what are their addresses) rather than component-specific. Nevertheless, IP-XACT standard defines address space inside a component, although it is seldom a reusable element.

Each address space has a mandatory unique name and an optional textual description.

Addressable unit bits specifies the number of bits each address increment contains, e.g. 8 bits or 32 bits. The AUB is mandatory if the address space contains a local memory map default AUB is 8.

Range (mandatory) means the size of the address space expressed as addressable units. In other words, it is the number of separate addresses.

Width (mandatory) is the data width of a row in bits, defining the maximum size for a single transfer.

Master interface binding ties the address space to a master interface. The binding can be set in the bus interface editor.

Address spaces editor contains a context menu (right mouse button) providing following options:

  • Add new address space (Add row)
  • Remove address space (Remove row)
  • Clear the selected cells
  • Copy the contents of the selected cells to clipboard
  • Paste the contents of the clipboard to the selected cells
  • Import a csv-file to the editor
  • Export the contents of the editor to a csv-file
